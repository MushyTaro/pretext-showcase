example.js 
import { prepare, layout } from '@chenglou/pretext'

// Step 1 — run once per text (segments + canvas measure)
const prepared = prepare(
  'AGI 春天到了. بدأت الرحلة 🚀',
  '16px Inter'
)

// Step 2 — pure arithmetic, called on EVERY resize
// No DOM. No reflow. ~0.0002ms per call.
const { height, lineCount } = layout(
  prepared,
  320,   // maxWidth in px
  24    // lineHeight in px
)

// → { height: 48, lineCount: 2 }
Under the hood

Two phases, one insight

The DOM Problem

Calling getBoundingClientRect() or reading offsetHeight forces the browser to synchronously flush all pending style/layout changes — a full layout reflow. 500 text blocks = 500 reflows = 30ms+ per frame.

expensive

prepare(text, font)

Segments text with Intl.Segmenter, measures each word width via Canvas measureText, and caches the results. Handles emoji correction, bidirectional levels, CJK per-char breaking. Runs once per text.

one-time cost

layout(prepared, w, lh)

Pure arithmetic over cached widths. Walks segments, sums widths, breaks lines — no DOM, no canvas, no side effects. Safe to call on every pixel of resize. ~0.0002ms per text.

near-zero cost

Shrink-wrap demo

Tightest-Fit Chat Bubbles

Each bubble's width is binary-searched using layout() to find the minimum maxWidth that keeps the same number of lines — eliminating the wasted whitespace CSS fit-content leaves behind. The badge shows the exact shrink-wrapped width.

Physics demo

Text Ripple Wave

Each word is measured with prepare() once, then positioned without touching the DOM. A 2D wave height map propagates across a grid — transform is the only property updated per frame. Move your mouse or click to make waves.

i18n support

Every Writing System

pretext handles CJK (character-level breaks), Arabic & Hebrew (RTL + bidi), Devanagari, Thai, Khmer, emoji sequences, and mixed bidirectional text — with emoji canvas/DOM correction built in.
